# 人类视频 → G1 视频：基于反向生成式数据合成的跨具身视频编辑

## 1. 问题定义

**目标:** 训练一个视频编辑模型，输入人类第三人称视频，输出对应的 Unitree G1 人形机器人第三人称视频。

---

## 2. 核心 Insight：反转数据合成方向

```
传统方向: 人类视频(真实) → 合成pipeline → 机器人视频(合成，有gap)
我们的方向: G1视频(真实) → 合成pipeline → 人类视频(合成，作为输入)
```

**关键优势：**
1. **输出端零 domain gap** — 训练目标是真实 G1 视频，输出质量有保证
2. **输入端容错** — 合成的人类视频作为输入端，即使不完美，模型也能学到正确映射；推理时真实人类视频质量更高，泛化更好（train-on-hard, test-on-easy）
3. **一对多增强** — 一段 G1 视频可合成 N 种不同外貌/服装/体型的人类视频，天然数据多样性
4. **只需机器人视频** — 不需要配对采集，不需要 3D 引擎，只需真实 G1 视频即可启动

---

## 3. 技术路线

### 阶段一：反向数据合成

```
真实G1视频 ──→ 选取关键帧 ──→ 图像编辑(G1全身→人类) ──→ 人类关键帧
    │                                                        ↓
    └──→ 提取控制信号(深度/边缘/分割) ──→ Cosmos Transfer ──→ 合成人类视频
                                           + 关键帧锚定

× N 种人类外观 → 输出: {(合成人类视频_i, 真实G1视频)} × N
```

- **关键帧编辑**：检测 G1 全身并 mask，用 Flux Fill inpaint 为人类；可控制外观多样性（肤色、服装、性别、体型）
- **视频合成**：Cosmos Transfer 2.5，以关键帧锚定外观 + 控制信号（深度/边缘/分割）保持运动对齐

### 阶段二：编辑模型训练

基于 Wan 2.2 (DiT + Flow Matching)，改造为 video-in/video-out：
- 输入：人类视频 → VAE encode → condition tokens
- 输出：G1 视频 → VAE encode → denoising target
- LoRA 微调，保留预训练能力

推理时输入互联网人类视频，批量生成 G1 视频数据集。

---

## 4. 与相关工作的关键区别

### vs X-Humanoid
| 维度 | X-Humanoid | Ours |
|------|-----------|------|
| 合成方式 | UE5 3D 渲染（两端都是合成） | **生成式合成（真实机器人视频为锚）** |
| 输出端质量 | 合成的（sim2real gap） | **真实的（零 domain gap）** |
| 数据多样性 | 受限于动画资产 | **一对多，外观可控** |
| 3D 资产依赖 | 需要精确 3D 模型 | **不需要** |

### vs Masquerade / H2R / Mitty
| 维度 | 现有方法 | Ours |
|------|---------|------|
| 合成方向 | 前向（人→机器人） | **反向（机器人→人）** |
| 机器人类型 | 固定臂 (Franka/UR5) | **人形机器人 (G1) 全身** |
| 输出端质量 | 合成的机器人视频 | **真实机器人视频** |
| 数据增强 | 1:1 | **1:N** |

---

## 5. 创新点

1. **反向生成式数据合成**：以真实机器人视频为锚反向合成，输出端零 domain gap
2. **一对多数据增强**：同一段机器人视频合成多种人类外观变体
3. **无 3D 资产的可扩展框架**：纯生成式，可快速迁移到其他机器人平台

---

## 6. 数据来源

**G1 视频（合成端）：** 自采集/合作实验室的 G1 第三人称视频，涵盖室内操作、行走、搬运等

**人类视频（推理端）：** 互联网第三人称人类活动视频

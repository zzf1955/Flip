## 背景

训练 pipeline 的预计算缓存散落在 `output/` 下多个临时命名目录中（`mitty_cache_1s`, `mitty_cache_robot`, `data_cache_80` 等），VAE latent 和 T5 embedding 混存在同一个 .pth 中导致 T5 被重复存储数千次（`mitty_cache_robot` 89GB 中约 75GB 是重复 T5），命名无规范。

## 目标

1. 在 `training_data/cache/` 下建立统一的 cache 目录结构
2. T5 embedding 全局共享（prompt 级别），VAE latent 按样本存储
3. cache 目录命名与源数据目录对应（`vae/pair_1s/` ↔ `pair/1s/`）
4. 所有训练脚本兼容新格式，旧格式自动 fallback

## 范围

**包含：**
- `config.py` 新增 cache 路径常量
- `mitty_cache.py` 重构：VAE 和 T5 分别输出
- `train_utils.py:load_sample()` 兼容新旧格式
- `train.py` / `train_mitty.py` / `train_rf.py` 新增 `--t5-cache-dir` 参数
- eval 视频帧按需加载（去掉 cache 中的 PIL frames）
- 文档更新

**不包含：**
- 旧 cache 的自动迁移（重新生成即可）
- `train_lora.py` (legacy DiffSynth pipeline，不改)
- `hand_patch` 位置变更（已在 `pair/<split>/hand_patch/` 下，保持不动）

## 实施计划（分步骤）

### Step 1: config.py 新增路径常量
- `CACHE_ROOT = os.path.join(TRAINING_DATA_ROOT, "cache")`
- `T5_CACHE_DIR = os.path.join(CACHE_ROOT, "t5")`
- `VAE_CACHE_DIR = os.path.join(CACHE_ROOT, "vae")`

### Step 2: mitty_cache.py 重构
- T5 embedding 单独输出到 `T5_CACHE_DIR/`：`<hash>.pth` + `negative.pth`
- VAE latent 不再包含 context_posi/context_nega/frames
- 新 .pth 格式：`{human_latent, robot_latent, prompt, source_id}`
- 保留 `--output` 参数，默认改到 `VAE_CACHE_DIR/<dataset>/<split>`

### Step 3: train_utils.py 兼容加载
- `load_sample()` 检测：有 `context_posi` → 旧格式直接用；无 → 从 T5 cache 注入
- 新增 `load_t5_cache(t5_dir)` → 返回 `{prompt: embedding}` 映射
- eval 帧：去掉 cache 中的 frames，改为 `load_eval_frames(source_id, pair_dir)`

### Step 4: 训练脚本适配
- `train.py` / `train_mitty.py` / `train_rf.py` 新增 `--t5-cache-dir` 参数
- 训练开始时一次性加载 T5 cache 到内存
- `generate_eval_videos()` 适配无 frames 的情况

### Step 5: 文档更新
- `doc/step_5_training_infra.md` 更新 cache 说明和命令
- `CLAUDE.md` 更新目录布局

## 验收标准

1. `mitty_cache.py` 生成的新格式 cache 中 VAE latent 与旧格式数值一致
2. `train.py --cache-train training_data/cache/vae/pair_1s/train --max-steps 5` 跑通
3. 旧格式 cache 仍可直接用于训练（向后兼容）
4. T5 cache 全局唯一，不再重复存储

## 测试要求

1. 生成小规模 cache 并验证 latent 一致性
2. 新旧格式各跑 5 步训练 smoke test，对比 loss 值
3. eval video 生成正常（无 frames 时从源视频读取）

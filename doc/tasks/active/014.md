## 背景

当前三个训练脚本的训练长度由 `--epochs × --repeat` 间接控制，不够直观。W&B 实验管理松散：项目名默认关闭、run 命名只有时间戳无法区分脚本来源、tag 没有记录关键超参。

## 目标

1. 用 `--max-steps` 直接控制训练步数，移除 `--epochs` / `--repeat`
2. W&B 项目默认 `"Flip"`，run 命名包含脚本前缀 + 关键参数 + 时间戳
3. W&B tag 自动从 args 提取关键超参（lora_rank, lr, batch_size, warmup, max_steps 等）

## 范围

- 包含：`train_utils.py`, `train_lora.py`, `train_mitty.py`, `train_rf.py`, `CLAUDE.md`
- 不包含：loss 函数、模型加载、eval 逻辑、DDP 同步

## 实施计划（分步骤）

1. `train_utils.py`: 新增 `infinite_file_batches()`, `build_run_name()`, `build_wandb_tags()`
2. 三个训练脚本：替换 epoch 循环为无限迭代器，更新 CLI 参数、run 命名、W&B tag
3. `CLAUDE.md`: 更新示例命令

## 验收标准

- `--max-steps N` 控制训练步数，数据无限循环
- W&B run 名如 `mitty-r96-lr1e-4-2026-04-20_153000`
- W&B tag 包含 `["mitty", "r96", "lr1e-4", "steps400", ...]`
- import 无误，dry-run 可通过

## 测试要求

- `infinite_file_batches` 确定性 + 分片正确性断言
- 单 GPU `--max-steps 3` smoke test

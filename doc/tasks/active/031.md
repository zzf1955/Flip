## 背景

当前仓库经历过多轮实验迭代，`src/pipeline/` 中同时保留 Mitty、RectFlow/FunControl、直接替换噪声、legacy LoRA 等多条训练路径，部分脚本互相 import 复用 legacy helper，导致训练入口、cache 口径、loss/patch 逻辑和文档说明都比较混乱。

用户明确当前项目主线为：

1. FunControl 相关部分不用保留；
2. 直接替换噪声实验已经废弃，不再维护；
3. 主要训练 pipeline 是直接外观替换（Mitty）和三阶段 LoRA 训练。

因此需要在独立 worktree 中进行一次较大幅度的训练代码重构，删除或隔离废弃路径，收敛正式入口和公共训练组件。

## 目标

1. 明确训练主线入口：Mitty 外观替换与三阶段 LoRA 训练为正式维护路径。
2. 清理 FunControl / RectFlow / 直接替换噪声等废弃实验入口和引用，避免后续误用。
3. 将 Mitty 与 LoRA 训练共享的加载、cache、日志、W&B、评估视频等逻辑沉淀为公共模块。
4. 更新训练相关文档，明确哪些入口保留、哪些实验废弃、推荐命令如何运行。
5. 保持数据生成与 inpaint / retarget pipeline 不做大范围行为变更，降低重构风险。

## 范围

**包含：**

- 盘点并标记训练相关入口：`train.py`、`train_mitty.py`、`train_rf.py`、`train_lora.py`、`mitty_cache.py`、`mitty_model_fn.py`、`rf_model_fn.py`、`backbones/` 等。
- 移除或归档 FunControl / RectFlow / 直接替换噪声相关代码路径和文档引用。
- 重构 Mitty 外观替换训练入口，使公共 helper 不再从 legacy 脚本互相 import。
- 梳理三阶段 LoRA 训练的入口、配置参数和文档描述。
- 更新 `doc/step_5_training_infra.md`、`doc/step_5_two_stage_training.md`、`doc/step_5_wan22_architecture.md` 或新增训练架构文档。
- 更新 `doc/requirement-log.md` 记录本任务。

**不包含：**

- 不重新设计 inpaint、SAM2、retarget、人形渲染算法。
- 不保留 FunControl 兼容层。
- 不维护直接替换噪声实验的 fallback 分支。
- 不迁移或自动转换历史 checkpoint / cache；必要时文档说明重新生成方式。
- 不在 main 分支直接开发，实际代码修改在 `.worktrees/t031` 中完成。

## 实施计划（分步骤）

### Step 1: 训练入口与依赖盘点

- 用静态搜索列出训练相关文件、CLI 参数、互相 import 关系。
- 区分正式入口、待删除入口、公共组件、仅文档引用。
- 形成重构前后入口表，作为文档和实现依据。

### Step 2: 公共训练组件下沉

- 新建或整理训练公共模块，承载 cache sample 加载、T5/VAE 读取、patch weight、eval video、W&B run/tag、LoRA checkpoint 等逻辑。
- 消除 `train.py` 从 `train_mitty.py` 反向复用 helper 的结构。
- 保持异常显式失败，不添加旧行为 fallback。

### Step 3: 清理废弃路径

- 移除或归档 FunControl / RectFlow / 直接替换噪声入口与 backbone 注册。
- 删除废弃入口在文档和命令示例中的推荐地位。
- 对确认为废弃的脚本，优先删除；若仍需参考，则移动到 archive 并标记不可运行。

### Step 4: 收敛正式训练入口

- Mitty 外观替换保留正式入口和 cache 生成路径。
- 三阶段 LoRA 训练保留正式入口，明确各阶段输入输出和参数。
- CLI 参数命名统一，错误组合使用 `argparse` 显式报错。

### Step 5: 文档和验证

- 更新训练 infra / 架构 / 任务文档，说明废弃内容和推荐流程。
- 执行 import smoke test 与不依赖大模型的单元/静态检查。
- 如环境允许，运行小规模参数解析或 dry-run 级验证；大模型训练 smoke test 可列为手动验证项。

## 验收标准

1. 代码中不再将 FunControl / RectFlow / 直接替换噪声作为正式训练路径暴露。
2. Mitty 外观替换与三阶段 LoRA 训练入口清晰，文档中有推荐命令和数据流说明。
3. 公共训练 helper 不再存放在 legacy 入口脚本中供其他入口反向 import。
4. `python -m` import smoke test 覆盖保留的训练相关模块并通过。
5. 文档明确列出废弃实验，避免后续继续维护或误跑。

## 测试要求

1. 运行训练相关模块 import smoke test。
2. 运行核心公共 helper 的轻量级单元/参数解析测试（如可行）。
3. 对保留入口执行 `--help` 或 parser 构建验证，确保 CLI 可用。
4. 如需大模型/数据的训练 smoke test，记录命令、依赖和未运行原因。

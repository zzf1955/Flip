## 背景

`retarget_video.py` 生成的 SMPLH overlay 视频中手指始终是静态姿态（拇指对掌 + 四指伸直），G1 hand_state 数据已加载但未用于驱动 SMPLH 手指。需要实现运行时映射以便调试手部关节对应关系。

## 目标

1. 新增 `apply_finger_curl_from_g1(hand_state)` 函数，将 G1 Inspire 12 维 hand_state 映射到 SMPLH 45 维 hand_pose
2. `retarget_video.py` 逐帧调用该函数，SMPLH 手指跟随 G1 同步变化
3. `retarget_diag.py` 增加 `--live-hand` 选项用于单帧调试

## 范围

**包含：**
- `src/core/retarget.py`: 新增常量 + `apply_finger_curl_from_g1()` 函数
- `src/pipeline/retarget_video.py`: 手姿态从静态改为逐帧计算
- `src/tools/retarget_diag.py`: 新增 `--live-hand` flag

**不包含：**
- BrainCo 手部映射（fallback 到默认姿态）
- 其他 pipeline（human_overlay, cosmos_prepare, segment_pipeline）的集成

## 实施计划（分步骤）

### Step 1: `retarget.py` 新增常量

在 `FINGER_CURL_SIGN_R` 后添加从 Inspire URDF 推导的角度范围常量：
- `INSPIRE_FINGER_CURL_PER_SEG_RAD` = (1.4381 + 1.4381*1.0843) / 3.0 ≈ 0.999 rad
- `INSPIRE_THUMB_CLOSE_PER_SEG_RAD` = Inspire 三级联拇指总角 / 2
- `INSPIRE_THUMB_TILT_EXTRA_DEG` = 15.0
- `G1_TO_SMPLH_FINGERS` = ["index", "middle", "ring", "pinky"]

### Step 2: `retarget.py` 新增 `apply_finger_curl_from_g1()`

在 `build_finger_curl_pose()` 之后添加。逻辑：
1. 反转+重排序 hand_state（同 build_q 逻辑）
2. 以 `build_default_hand_pose()` 为基底
3. 四指: closed_ratio × PER_SEG_RAD → 3 段均分, Z 轴, L/R 符号
4. 拇指 close → thumb2/thumb3; tilt → thumb1 叠加对掌轴

### Step 3: `retarget_video.py` 集成

1. import `apply_finger_curl_from_g1`
2. 删除 loop 外静态 hand_pose
3. loop 内逐帧调用 `apply_finger_curl_from_g1(hs, hand_type)`

### Step 4: `retarget_diag.py` 添加 `--live-hand`

1. import `apply_finger_curl_from_g1`
2. 添加 `--live-hand` argparse flag
3. 在 dispatch 逻辑中增加 `elif args.live_hand` 分支

### Step 5: 验证

单帧诊断 + 视频渲染，对比有/无 live-hand 差异。

## 验收标准

1. `apply_finger_curl_from_g1(np.zeros(12))` 返回全闭合手指姿态（非零值）
2. `apply_finger_curl_from_g1(np.ones(12))` 返回全张开手指姿态（四指为零）
3. BrainCo hand_type 正确 fallback 到默认姿态
4. `retarget_video.py` 输出的 SMPLH 面板手指随 G1 同步变化
5. `retarget_diag.py --live-hand` 生成诊断图

## 测试要求

- 单元级：验证全开/全闭/中间值的输出数值正确性
- 视觉验证：`retarget_diag.py --live-hand` 单帧 + `retarget_video.py` 视频

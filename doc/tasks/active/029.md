## 背景

两阶段训练已完成：Phase 1 在 attention (q,k,v,o) 上训练 identity LoRA，Phase 2 merge identity LoRA 后在 FFN (ffn.0,ffn.2) 上训练外观 LoRA。下一阶段需要同时 merge 两个 LoRA（identity + FFN），重新在 attention 上加 LoRA 训练外观替换。当前 `--merge-lora` 只接受一个路径，无法满足需求。

## 目标

1. `--merge-lora` 支持多次指定，可合并多个 LoRA checkpoint
2. 用 124 个 human→robot pair 样本在 attention LoRA 上训练外观替换

## 范围

- **包含**：`src/pipeline/train.py` CLI 参数 + 传参逻辑；`src/pipeline/train_mitty.py` 多 LoRA 合并
- **不包含**：`train_rf.py`（不支持 merge-lora）；训练数据准备

## 实施计划（分步骤）

1. `train.py`：`--merge-lora` 改为 `action="append"` 支持多路径
2. `train_mitty.py`：`MittyTrainingModule.__init__()` 循环调用 `merge_lora_into_weights()`
3. `train_mitty.py` 自身 CLI 同步更新
4. 验证：单卡 + 多卡 smoke test

## 验收标准

- `--merge-lora a.safetensors --merge-lora b.safetensors` 两个都被 merge
- 日志打印合并的 LoRA pairs 总数
- 单个 `--merge-lora` 仍然正常工作（向后兼容）
- DDP 多卡正常

## 测试要求

- 单卡 smoke test：`--max-steps 5 --eval-steps 5` + 双 merge-lora
- 多卡 smoke test：`torchrun --nproc_per_node=2 --max-steps 5 --eval-steps 5`

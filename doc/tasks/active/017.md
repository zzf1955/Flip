## 背景

训练 pipeline 中多个加载路径仍经过 CPU 中转再拷贝到 GPU：VAE 权重、训练样本 cache、patch weights、init LoRA checkpoint、mitty_cache.py 中 T5 文本编码器（使用 DiffSynth CPU offload）。DiT 直接 GPU 加载已在 t014b 合并完成。

## 目标

1. 所有模型/数据加载路径直接读到目标 GPU，不经过 CPU 中转
2. mitty_cache.py 彻底脱离 DiffSynth 的 `WanVideoPipeline.from_pretrained()`，使用 wan_loader.py 的直接加载器
3. 训练脚本的 prefetch / eval / init_lora 全部传 device

## 范围

**包含：**
- `wan_loader.py`: load_vae 透传 device；新增 load_text_encoder + load_tokenizer
- `train_utils.py`: load_sample 新增 device 参数
- `train_mitty.py` / `train_rf.py` / `train.py`: 全路径 GPU 直接加载
- `mitty_cache.py`: 彻底脱离 DiffSynth，使用直接加载器

**不包含：**
- `train_lora.py`（Wan 2.1 legacy，DiffSynth 内部管线）
- Loss 函数、前向逻辑、DDP 同步
- WandbLogger / CsvLogger

## 实施计划（分步骤）

### Step 1: wan_loader.py — VAE 直接 GPU 加载
- `load_vae()` 透传 home_device 到 `_read_safetensors` 和 `torch.load`
- `WanVideoVAEStateDictConverter` 中 dtype 转换改为 `v.to(dtype=dtype)` 显式指定

### Step 2: train_utils.py — load_sample 支持 device
- 新增 `device: str = "cpu"` 参数

### Step 3: 训练脚本全路径 GPU 直接加载
- VAE: `home_device="cpu"` → `home_device=device`
- prefetch: `load_sample(f)` → `load_sample(f, device=args.device)`
- `_load_patch_weights`: 新增 device 参数
- eval_loss / generate_eval_videos: load_sample 传 device
- init LoRA: `load_file(path)` → `load_file(path, device=str(self.device))`

### Step 4: wan_loader.py — 新增 T5 直接加载器
- `load_text_encoder(path, device, dtype)`: WanTextEncoder 直接加载
- `load_tokenizer(tokenizer_dir, seq_len)`: HF AutoTokenizer

### Step 5: mitty_cache.py — 彻底脱离 DiffSynth
- 删除 `WanVideoPipeline` / `ModelConfig` import
- 使用 wan_loader 的 load_text_encoder / load_vae / load_tokenizer
- 实现独立的 encode_prompt / encode_video

### Step 6: 验证
- 单 GPU smoke test (train_mitty --max-steps 3)
- mitty_cache.py 编码 2-3 个样本，确认输出格式兼容

## 验收标准

1. `nvidia-smi` / 日志确认所有模型直接加载到 GPU，无 CPU 中转
2. 训练 smoke test 通过（3 步无 OOM，loss 正常）
3. mitty_cache.py 输出 .pth 与旧版格式兼容（key 一致、shape 一致）
4. DiffSynth 的 `WanVideoPipeline` 在 mitty_cache.py 中完全移除

## 测试要求

1. 单 GPU smoke test: `CUDA_VISIBLE_DEVICES=1 python -m src.pipeline.train_mitty --cache-train ... --max-steps 3 --save-steps 2 --eval-steps 2`
2. mitty_cache.py: 编码少量样本对比输出

## 背景

`src/pipeline/train_mitty.py:115` 的 `build_pipe` 走 `WanVideoPipeline.from_pretrained` 加载 Wan2.2 TI2V-5B DiT + VAE，慢到 DDP 必须错峰加载（CLAUDE.md 已列为硬规则）。实测瓶颈来自 DiffSynth 为通用 plug-and-play 设计的抽象层，对我们固定模型 + 固定显存布局的训练场景完全是 overhead：

1. `hash_model_file`（`DiffSynth-Studio/diffsynth/core/loader/file.py:126`）扫描每个 safetensors 分片的全部 key+shape 算 md5，在 `MODEL_CONFIGS` 表里反查 variant。
2. `DiskMap` 再扫一遍 metadata 建 `{key→(file,offset)}` 索引。
3. `enable_vram_management` 把每个 Linear 替换成 `AutoWrappedLinear`（`core/vram/layers.py:429`），即使 `offload_device==onload_device==cuda` 也包一层 forward 状态机。
4. `WanVideoPipeline.__init__` 实例化 25 个推理用 `PipelineUnit`，训练一个都不走。

## 目标

- 新建直给式 loader，DiT 冷启动加载时间 ≥30% 下降；DDP 4 rank 启动总时间可观下降。
- DiT forward 不再过 `AutoWrappedLinear`。
- `train_mitty.py` 的 `MittyTrainingModule`、`generate_eval_videos`、LoRA 保存等接口零改动。
- LoRA ckpt 的 key 集合与替换前完全一致。

## 范围

**包含**：
- `src/core/wan_loader.py`（新建）：DiT/VAE 直接加载 + `SimplePipe` 最小接口
- `src/pipeline/train_mitty.py::build_pipe`（函数体替换）

**不包含**：
- `src/pipeline/mitty_cache.py`（T5 路径，一次性开销，本次不碰）
- `src/pipeline/train_lora.py`（走 DiffSynth `WanTrainingModule`，另一条路径）
- 推理 / 生成脚本（`wan_regen.py` 等）

## 实施计划（分步骤）

### 1. `src/core/wan_loader.py`（新建）

DiT 配置 vendored 自 `DiffSynth-Studio/diffsynth/configs/model_configs.py:290-295`：

```python
WAN22_TI2V_5B_DIT_CONFIG = {
    'has_image_input': False, 'patch_size': [1, 2, 2], 'in_dim': 48,
    'dim': 3072, 'ffn_dim': 14336, 'freq_dim': 256, 'text_dim': 4096,
    'out_dim': 48, 'num_heads': 24, 'num_layers': 30, 'eps': 1e-06,
    'seperated_timestep': True, 'require_clip_embedding': False,
    'require_vae_embedding': False, 'fuse_vae_embedding_in_latents': True,
}
```

- `load_dit(shard_paths, device, dtype=torch.bfloat16) -> WanModel`
  - `skip_model_initialization()` 下 `WanModel(**WAN22_TI2V_5B_DIT_CONFIG)` 建空壳
  - `ThreadPoolExecutor` 并发 `safe_open` + `get_tensor` 读分片
  - `load_state_dict(sd, assign=True)` → `.to(device, dtype).eval()`

- `load_vae(path, device, dtype=torch.bfloat16, home_device="cpu") -> WanVideoVAE38`
  - `WanVideoVAE38(z_dim=48, dim=160)` 空壳
  - `.pth`：`torch.load` → `WanVideoVAEStateDictConverter`
  - `.safetensors`：直接读
  - `.to(home_device, dtype).eval()`

- `class SimplePipe(nn.Module)`
  - `.device`、`.dit`、`.vae`、`.scheduler = FlowMatchScheduler("Wan")`、`.model_fn`
  - `load_models_to_device(names)`：`"vae"` → `.to(device)`；其它 no-op

### 2. `src/pipeline/train_mitty.py::build_pipe`

函数体换成调用 `src.core.wan_loader`，签名不变，`tokenizer_dir` 参数接收但忽略（调用方已传）。

### 3. DDP 错峰保留

`train_rank` barrier 循环保留（40GB 峰值 CPU RAM 的规避仍然需要）。

## 验收标准

1. **功能对等**：`python -m src.pipeline.train_mitty --cache-train output/mitty_cache_1s/train --cache-eval output/mitty_cache_1s/eval --cache-ood output/mitty_cache_1s/ood_eval --epochs 1 --repeat 1 --save-steps 5 --eval-steps 5 --eval-video-steps 10` 完整走完 10 step，loss 数量级与替换前一致。
2. **加载速度**：`time python -c "from src.pipeline.train_mitty import build_pipe; build_pipe(...)"` 冷启动缩短 ≥30%。
3. **DDP 启动**：`torchrun --nproc_per_node=4 -m src.pipeline.train_mitty ...` 正常训练，无 CPU OOM。
4. **ckpt 一致**：`step-0005.safetensors` 的 key 集合与替换前完全一致。

## 背景

DiffSynth-Studio 的训练框架缺陷严重：
- 不输出 train loss（计算后被丢弃）
- 无 train/eval 数据集分割
- 无任何日志系统（无 TensorBoard / WandB / print）
- 只有 tqdm 进度条 + checkpoint 文件

已跑通两阶段 baseline（阶段 1 embedding 缓存 + 阶段 2 LoRA 训练），但无法观察训练过程。

## 目标

新建 `src/pipeline/train_lora.py`，复用 DiffSynth 的模型加载和 forward pass，自写训练循环，支持：
1. Train / Eval 分割（固定 seed 可复现）
2. 每步输出 train loss
3. 定期计算 eval loss
4. 训练产物（ckpt、日志、eval 视频占位）按日期归档到 `training_data/log/`

## 范围

**包含：**
- `src/pipeline/train_lora.py` — 新训练脚本
- 单卡训练（不用 accelerate DDP），先跑通再扩展
- 读取已有 data_cache .pth 缓存（阶段 1 的输出）
- 文本日志输出到 `training_data/log/<date>/train.log`
- Checkpoint 保存到 `training_data/log/<date>/ckpt/`

**不包含：**
- 不改 DiffSynth-Studio 代码
- 不实现 eval 视频生成（目录预留）
- 不实现 DDP 多卡（后续任务）
- 不改阶段 1 data_process 流程

## 实施计划（分步骤）

### Step 1: 数据加载 + Train/Eval 分割

- 扫描 cache 目录 `*.pth`，torch.load 获取 (inputs_shared, inputs_posi, inputs_nega)
- 按 `--eval-ratio`（默认 0.1）随机分割，固定 `--seed 42`
- 80 样本 → 72 train + 8 eval

### Step 2: 模型初始化

- 复用 DiffSynth 的 `WanTrainingModule`（import from train.py）
- 传入相同参数：task="sft:train", fp8, lora_rank=16, gradient_checkpointing
- 单卡 `--device cuda:3`

### Step 3: 训练循环

- AdamW optimizer，ConstantLR scheduler
- 每步调用 `model.forward(data={}, inputs=batch)` 获取 loss
- 打印 + 写入日志：step, train_loss, lr, 耗时
- 按 `--save-steps` 保存 LoRA checkpoint（safetensors 格式）

### Step 4: Eval 循环

- 按 `--eval-steps` 间隔计算 eval loss
- torch.no_grad()，遍历 eval 集取均值
- 输出到日志

### Step 5: 输出目录

```
training_data/log/YYYY-MM-DD_HHMMSS/
├── ckpt/step-NNN.safetensors
├── eval/                          ← 预留
└── train.log
```

## 验收标准

1. `python -m src.pipeline.train_lora --cache-dir output/data_cache_80 --device cuda:3 --epochs 1 --repeat 1 --save-steps 5 --eval-steps 5` 能正常运行
2. `train.log` 每步有 train_loss 数值输出（不是 NaN）
3. eval_loss 在指定间隔输出
4. checkpoint 保存为 .safetensors
5. 单步耗时与 DiffSynth baseline 相当（~6s）

## 测试要求

- 用 `output/data_cache_80`（80 个缓存样本）跑 10 步验证 pipeline 通过
- 确认 loss 值合理（不是 0 / NaN / 极大值）
- 确认 checkpoint 文件可被 DiffSynth 推理加载

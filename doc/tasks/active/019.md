## 背景

统一训练入口 `train.py` 目前用 `--epochs` + `--repeat` 间接计算 total_steps（`epochs × steps_per_epoch`），用户无法精确控制训练步数，且 epoch 边界会导致尾部 batch 被丢弃、实际步数与预期不一致。

legacy 脚本 `train_mitty.py` / `train_rf.py` 已经是纯 step-based（`--max-steps` + `infinite_file_batches`），接口不统一。

## 目标

`train.py` 改为纯 step-based 控制：用户直接指定 `--max-steps`，去掉 `--epochs` / `--repeat`，训练步数精确可控。

## 范围

**包含：**
- `src/pipeline/train.py` — argparse、total_steps 计算、训练循环、日志
- `CLAUDE.md` — 更新示例命令

**不包含：**
- `train_mitty.py` / `train_rf.py`（已经是 step-based，不改动）
- `train_utils.py` 中的 `infinite_file_batches`（直接复用，不改动）
- eval/save/log/LR scheduler 逻辑（已经是 step-based，不改动）

## 实施计划（分步骤）

### Step 1: argparse 参数变更

- 删除 `--epochs`（line 541）和 `--repeat`（line 542）
- 新增 `--max-steps`（int, required）

### Step 2: total_steps 计算简化

删除 lines 297-300 的 epoch_samples/samples_per_rank/steps_per_epoch 计算，改为：
```python
total_steps = args.max_steps
```

### Step 3: 训练循环重写

删除 `for epoch in range(args.epochs):` 外层循环（lines 341-479）及其内部的手动 batching。

改为 legacy 风格：
```python
data_iter = infinite_file_batches(
    train_files, batch_size=args.batch_size,
    world_size=world_size, rank=rank, seed=args.seed,
)
# _prefetch 定义一次
prefetch_ex = ThreadPoolExecutor(max_workers=1)
next_batch = next(data_iter)
pending = prefetch_ex.submit(_prefetch, next_batch)

while step < total_steps:
    samples = pending.result()
    next_batch = next(data_iter)
    pending = prefetch_ex.submit(_prefetch, next_batch)
    step += 1
    # ... 内部训练逻辑不变
```

### Step 4: 日志更新

删除 epoch 相关日志（lines 323-331），改为：
```python
info(f"Plan: {len(train_files)} train files, {total_steps} steps, effective_bs={effective_bs}")
```

### Step 5: 确认 import

确认 `train.py` 从 `train_utils` 导入了 `infinite_file_batches`，若没有则添加。

### Step 6: CLAUDE.md 更新

更新「统一训练入口」示例命令中的 `--epochs 3 --repeat 5` → `--max-steps 400`。

## 验收标准

1. `train.py` 不再有 `--epochs` / `--repeat` 参数
2. `--max-steps N` 能精确控制训练步数（跑 N 步后停止）
3. 数据通过 `infinite_file_batches` 无限循环，shuffle 确定性（DDP 安全）
4. LR cosine annealing 正确走完 max_steps
5. 恒等训练和外观替换训练均正常工作

## 背景

当前 Mitty 训练的恒等映射（R2R）效果不好。核心假设：5 帧 clean context 与预训练的 1 帧 clean 分布偏移太大；从纯高斯噪声重建 robot 视频路径太长，LoRA 小容量难以拟合。

方案 A（Rectified Flow）：去掉 Mitty concat，把 source latent 替代高斯噪声作为 flow 起点。模型学习 source→target 直线插值速度场，推理从 source 出发积分到 target。

## 目标

- 新增 `rf_model_fn.py` 和 `train_rf.py`，作为 Mitty 的对比实验
- 复用同一 cache 格式（mitty_cache），无需重新编码
- 支持双卡 DDP 训练
- 提供和 Mitty 相同的 eval 框架（loss + video 生成）

## 范围

**包含：**
- `src/pipeline/rf_model_fn.py` — RF forward + loss
- `src/pipeline/train_rf.py` — RF 训练循环

**不包含：**
- 不修改 Mitty 代码
- 不修改 cache 格式
- 不修改 train_utils.py

## 实施计划（分步骤）

1. 创建 `src/pipeline/rf_model_fn.py`
   - `rf_model_fn_wan_video()` — 统一 timestep 的 DiT forward
   - `RFFlowMatchLoss()` — source-as-noise 的 flow-match loss
2. 创建 `src/pipeline/train_rf.py`
   - 从 train_mitty.py fork，替换 Module/eval/标识
   - import 共享函数（build_pipe, prepare_sample, collate_batch 等）
3. 冒烟测试（单卡 + 双卡 DDP）

## 验收标准

1. 单 GPU 冒烟：loss 正常下降，无 NaN/OOM
2. eval video：从 source 出发能看到逐步变化
3. DDP 2 卡：梯度同步正确
4. 和 Mitty 用同一份 cache 数据可对比

## 测试要求

- 单卡 smoke test：1 epoch, repeat 1, save/eval 每 5 步
- 双卡 DDP test：torchrun --nproc_per_node=2

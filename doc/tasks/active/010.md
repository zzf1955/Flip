## 背景

当前数据 pipeline 是多个独立脚本串联（segment_episodes → seedance_batch → make_pair），存在以下不足：
1. 只有一条 Seedance 生成线（robot → Seedance → human），缺少利用 inpaint + SMPLH overlay 增强的路径
2. 手部 patch 权重图生成（hand_patch.py）独立于 make_pair，需要额外步骤和 mitty_cache 作为输入
3. 切片过程中中间数据过多

## 目标

1. 新增 `seedance_advance.py`：overlay 视频 → Seedance API → 增强人类视频，输出到 `training_data/seedance_advance/`
2. 新增 `hand_patch_4s.py`：在 4s segment 级别预计算 per-frame 手部 FK bbox，存为 parquet
3. 重构 `make_pair.py`：支持 seedance_advance 源 + 内联生成 hand patch 权重图

## 范围

**包含：**
- `src/core/config.py`: 新增 `SEEDANCE_ADVANCE_DIR`, `HAND_PATCH_4S_DIR`
- `src/pipeline/seedance_advance.py`: 新文件，overlay → Seedance 批量处理
- `src/pipeline/hand_patch_4s.py`: 新文件，4s segment per-frame 手部 bbox
- `src/pipeline/make_pair.py`: +seedance_advance 源, +hand patch 整合
- `src/pipeline/hand_patch.py`: `load_fk_model` 加 hand_type 参数

**不包含：**
- segment_episodes.py 改动
- segment_pipeline.py 改动
- seedance_batch.py 改动
- 训练代码（train_mitty.py 等）的适配

## 实施计划（分步骤）

### Step 1: config.py 新增常量

在 `OVERLAY_4S_DIR` 之后添加 `SEEDANCE_ADVANCE_DIR` 和 `HAND_PATCH_4S_DIR`。

### Step 2: seedance_advance.py 新文件

复用 seedance_gen.py 的 API 函数和 seedance_batch.py 的批量模式。

数据流：overlay/4s/ → upscale → Seedance API → seedance_advance/4s/

Prompt: CG→真实增强风格。

### Step 3: hand_patch_4s.py 新文件

复用 hand_patch.py 的 FK 逻辑，逐帧计算手部 bbox，存为 parquet。

输出：hand_patch/4s/{task}/ep{N}/seg{M}_hands.parquet

### Step 4: hand_patch.py 微调

load_fk_model 加 hand_type 参数，保持向后兼容。

### Step 5: make_pair.py 整合

- HUMAN_SOURCE_MAP 新增 seedance_advance
- 新增 --hand-patch / --hand-weight 参数
- 新增 load_hand_patch_data + compute_clip_weight_map 函数
- process_pair 中内联生成 hand_patch .pth

## 验收标准

1. `seedance_advance.py --task X --episode 0` 能成功调用 Seedance API 并输出到正确目录
2. `hand_patch_4s.py --task X` 生成的 parquet 包含正确的 per-frame bbox 数据
3. `make_pair.py --human-source seedance_advance --hand-patch` 输出的 pair 目录包含 hand_patch/*.pth
4. 每个 .pth 的 weights shape = (5, 30, 40)，手部区域权重 = 指定值，背景 = 1.0
5. 不带 --hand-patch 的 make_pair 行为完全不变

## 测试要求

- seedance_advance: 至少 1 个视频端到端 API 调用成功
- hand_patch_4s: 验证 bbox 数值合理性（在图像范围内、手可见帧非 NaN）
- make_pair: 对比有/无 --hand-patch 输出，确认 video/control_video 不受影响

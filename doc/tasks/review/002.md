## 目标

构建 Flux Fill inpainting 的 ComfyUI workflow 脚本，通过 REST API 调用 ComfyUI (port 8000) 对 LEVERB 帧进行图像编辑。

## 范围

- 编写 `scripts/comfyui_flux_inpaint.py`
- 复用 ComfyUIClient（上传图片、提交 workflow、轮询结果、下载输出）
- 支持 mask 和无 mask（全白）两种模式
- 对样本帧运行验证

## 实施计划（分步骤）

1. 参考 `archive/3-30-data_search_and_premtive_experiment/scripts/comfyui_inpaint.py` 编写新脚本
2. 提取 ComfyUIClient 类（上传、提交、轮询、下载）
3. 构建 Flux Fill workflow：LoadCheckpoint → CLIP Encode → Inpaint Conditioning → KSampler → VAE Decode → Save
4. 无 mask 模式：自动生成与输入图同尺寸的全白 mask
5. 默认参数：steps=28, cfg=1.0, denoise=0.85, guidance=30.0
6. Prompt: `"a human performing the same action, realistic, third person view, photorealistic"`
7. CLI 接口：`--image`, `--mask`(可选), `--output-dir`, `--prompt`, `--steps`, `--denoise`
8. 输出到 `data/leverb_edited/flux/`

## 验收标准

- 脚本可对单张帧执行 Flux Fill 编辑并保存结果
- 支持自定义 prompt 和参数
- 输出图像与输入同尺寸

## 测试要求

- 对 Task 001 提取的帧运行，ComfyUI 无报错
- 输出图像可正常查看
- 编辑效果基本合理（机器人区域有变化）

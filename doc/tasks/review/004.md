## 目标

编写自动化脚本，批量将 LEVERB 机器人帧替换为人类。使用 GroundingDINO 自动检测机器人方框位置生成 mask，配合 SDPose + ControlNet 控制人物姿态，通过 Flux Fill inpaint 生成人类替换图。

## 范围

- 新建 `scripts/batch_robot2human.py`
- 处理 `data/leverb_frames/` 下所有 PNG 帧
- 每帧输出 5 张图到 `data/leverb_edited/batch/<name>/`：
  - `original.png` — 480P 原图
  - `bbox.png` — 原图 + 检测框可视化
  - `mask.png` — 方框区域白色 mask
  - `pose.png` — SDPose 骨骼姿态图
  - `result.png` — 替换为人的结果

## 实施计划（分步骤）

1. 检查 GroundingDINO 依赖，安装缺失包和下载模型权重
2. 编写 `scripts/batch_robot2human.py`，包含：
   - `detect_robot_bbox()` — GroundingDINO 检测机器人位置
   - `create_bbox_mask()` — bbox + padding → 白色方框 mask
   - `draw_bbox_on_image()` — 在原图上画绿色检测框 + 置信度
   - `build_controlnet_pose_workflow()` — 构建 ComfyUI API prompt 格式的 workflow
   - `process_single_frame()` — 单帧完整处理流程
   - `main()` — CLI + 批量处理循环
3. ComfyUI workflow 参数沿用验证过的设置：
   - KSampler: ddim, ddim_uniform, 16 steps, cfg=1, denoise=0.85
   - ControlNet strength=0.8, type=openpose
   - prompt: clip_l="a human", t5xxl="a human is walking"
4. 图片 resize 到 856×480（480P, 16:9, 可被 8 整除）
5. 单帧测试验证
6. 批量运行 12 帧

## 验收标准

- 脚本 `python scripts/batch_robot2human.py` 可一键批量处理
- 每帧输出 5 张图，文件命名和目录结构正确
- bbox 检测正确框住机器人区域
- mask 为干净的白色方框（非 mask 区域为黑色）
- pose 姿态图可识别骨骼关键点
- result 中机器人被替换为人类，背景保留

- 对全部 12 帧批量运行，无报错

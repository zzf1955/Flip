## 背景

Stage 1 robot 重建训练中发现严重的 IO 瓶颈：每个 .pth 缓存文件 55MB，但训练实际只用 ~9MB 的 tensor 数据（human_latent + robot_latent + context_posi），剩余 ~46MB 是 PIL Image 帧（human_frames + robot_frames），仅在 eval video 生成时使用。4 卡 DDP 每步从 HDD 读 440MB，实际只用 41MB，**IO 浪费率 90%**。GPU 有效利用率仅 ~50%。

同时 eval 配置不够合理：eval-video-steps 和 eval-steps 独立设置导致不对齐，eval 集 75 个样本偏多。

## 目标

1. 剥离 train 缓存中的 PIL 帧，文件从 55MB 降到 ~9MB，IO 量降 6 倍
2. eval 集缩小到 50 个样本
3. eval video 生成频率与 eval loss 对齐，每次输出 4 条视频
4. 确保 W&B 完整记录 eval loss + eval videos

## 范围

**包含：**
- `src/pipeline/mitty_cache.py`：添加 `--no-frames` 选项
- `src/pipeline/train_mitty.py`：eval 对齐 + max-eval-files + prefetch 保留
- 一次性脚本剥离现有 train 缓存的 PIL 帧
- W&B eval video 上传确认可用

**不包含：**
- 缓存格式迁移到 safetensors（后续优化）
- eval 数据重新生成（用 --max-eval-files 截断现有 75 个）
- 多进程 prefetch（当前单线程 prefetch + 小文件已足够）

## 实施计划（分步骤）

### Step 1: `mitty_cache.py` — 添加 `--no-frames`

在 argparse 添加：
```python
ap.add_argument("--no-frames", action="store_true",
                help="skip saving PIL frames (human_frames/robot_frames) to reduce file size")
```

在 `data` dict 构建处，条件跳过 PIL 帧：
```python
if not args.no_frames:
    data["human_frames"] = human_frames
    data["robot_frames"] = robot_frames
```

### Step 2: `train_mitty.py` — 四处修改

**2a. eval-video 与 eval 对齐**
- 修改 `--eval-video-steps` 默认值描述：当值为 -1 时跟随 `--eval-steps`
- 在参数解析后：`if args.eval_video_steps == -1: args.eval_video_steps = args.eval_steps`
- 默认值改为 -1

**2b. max-eval-files**
- 新增 `--max-eval-files`（默认 50）
- 加载 eval_files 后截断：`eval_files = eval_files[:args.max_eval_files]`

**2c. eval-video-samples 默认值**
- `--eval-video-samples-in-task` 默认从 2 改为 4

**2d. W&B 确认**
- `--wandb-log-videos` 已默认 True，无需改动
- eval-video 对齐后，每次 eval 自动触发视频生成 + W&B 上传

### Step 3: 剥离现有 train 缓存的 PIL 帧

写一次性 Python 脚本 `scripts/strip_cache_frames.py`：
```
python scripts/strip_cache_frames.py output/mitty_cache_robot/train/
```
- 遍历所有 .pth，删除 human_frames/robot_frames，重新保存
- eval 缓存**不动**（eval video 需要 PIL 帧）
- 预计文件从 55MB → ~9MB，总量 400GB → ~67GB

### Step 4: 验证

1. 剥离后检查文件大小：`du -sh output/mitty_cache_robot/train/`
2. 重启训练，确认 step time 显著下降
3. 确认 eval loss + eval video 在同一 step 触发
4. 确认 W&B 能看到 eval video

## 验收标准

1. train 缓存文件 avg ≤ 12MB（从 55MB 降下来）
2. 训练 step time ≤ 3.5s（从 4.2s 降下来）
3. eval video 生成频率 = eval loss 频率
4. 每次 eval 输出 4 条视频到 W&B
5. eval 集合大小 = 50

## 测试要求

- mitty_cache.py --no-frames 跑 5 个样本，检查输出无 PIL 帧
- strip 脚本跑完后 load 验证 tensor 完整
- 重启训练跑 100 步，观察 step time 和 W&B 面板

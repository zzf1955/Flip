## 背景

T004 搞清楚 Wan 2.2 TI2V-5B 的 `seperated_timestep + fuse_vae_embedding_in_latents` 机制后，本任务实现 Mitty 的序列拼接 + 部分加噪 + 部分 loss 训练流程。

**复用 DiffSynth 的深度**：看完 `diffsynth/diffusion/loss.py:FlowMatchSFTLoss` + `wan_video.py:model_fn_wan_video`，发现 DiffSynth 已经做了 Mitty 方案所需的一大半：
- `FlowMatchSFTLoss` 里自带 `if "first_frame_latents" in inputs: latents[:,:,0:1] = first_frame_latents`，并在算 loss 时 `noise_pred = noise_pred[:,:,1:]` 跳过第 1 帧
- `model_fn_wan_video:1376-1380` 根据 `seperated_timestep + fuse_vae_embedding_in_latents` 自动构造 `[0]*第1帧 + [t]*剩余帧` 的 per-patch timestep
- **Mitty 方案 = 把"第 1 帧 clean"扩展成"前 f_H 帧 clean"**

实际改动只有两处：
1. 自写 `MittyFlowMatchLoss`（~30 行，基于 `FlowMatchSFTLoss` 改，把 first_frame 从 1 帧扩展成 f_H 帧）
2. Monkey-patch `model_fn_wan_video:1376-1380` 那 5 行 timestep 构造（`latents.shape[2]-1` 改成 `latents.shape[2]-f_H`）

其他**全部复用 DiffSynth**：WanVideoPipeline 加载、LoRA 注入（`q,k,v,o` rank 96）、VAE、T5、checkpoint、grad checkpointing。

与现有 `src/pipeline/train_lora.py` 的区别：
- 骨干：Wan 2.2 TI2V-5B（非 FunControl）
- 条件化：不走 FunControl 的 ref_conv / y 通道，而是把 human 视频 latent 直接**沿 temporal dim 拼到 robot latent 前**（复用 `first_frame_latents` 机制）
- Timestep：human 段所有 patch 用 t=0（clean），robot 段用 sampled t（已由 monkey-patch 的 model_fn 自动构造）
- Noise：只加到 robot 段 latent
- Loss：只在 robot 段计算
- LoRA：rank 96（对齐 Mitty），target `q,k,v,o`（不训 ffn，对齐 Mitty 图 2）
- 精度：bf16（不量化，5B + 4090 足够宽松。实测 DiT ~10GB + LoRA 0.6GB + 激活 3-5GB + VAE 0.5GB ≈ 15-18GB）

## 目标

1. 新写 `src/pipeline/mitty_cache.py`：两段视频 VAE encode → 缓存 (human_latent, robot_latent, text_emb)
2. 新写 `src/pipeline/train_mitty.py`：读 cache，构造 concat latent + per-frame timestep + LoRA，跑训练循环
3. 复用 `training_data/pair/1s/{train,eval,ood_eval}` 三个 split（已由 T004 后的数据重组完成，共 train 143 / eval 5 / ood_eval 8）
4. **DDP 4 卡** 跑通（等效 bs=4，对齐 Mitty 论文），产出 LoRA checkpoint + 两个 eval 曲线（in-task + OOD）

## 实施计划（分步骤）

### Step 1: 数据缓存 `mitty_cache.py`

- CLI：`--pair-dir training_data/pair/1s/<split> --output output/mitty_cache_1s/<split>`
- 遍历 `<pair-dir>/metadata.csv`
- 对每条样本：
  - 加载 human (`<pair-dir>/control_video/pair_NNNN.mp4`) 和 robot (`<pair-dir>/video/pair_NNNN.mp4`) 各 17 帧视频
  - 两段分别经 Wan2.2_VAE encode → `(1, 48, 5, h/16, w/16)`（TI2V-5B 的 VAE latent channels 是 48，T 压缩 4）
  - T5 encode prompt → `context` (正负样本)
  - 保存为 `<output>/NNNN.pth`：`{"human_latent": ..., "robot_latent": ..., "context_posi": ..., "context_nega": ..., "source_id": ...}`
- 分别对 `train`, `eval`, `ood_eval` 三个 split 各跑一次

### Step 2a: Patched model_fn `mitty_model_fn.py`

- 复制 `diffsynth/pipelines/wan_video.py:model_fn_wan_video` 为 `mitty_model_fn_wan_video`
- 改 1376-1380 的 timestep 构造：
  ```python
  # 原版：只让第 1 帧 clean
  torch.zeros((1, h * w // 4)),
  torch.ones((latents.shape[2] - 1, h * w // 4)) * timestep
  # Mitty 版：让前 f_H 帧都 clean
  torch.zeros((f_H, h * w // 4)),
  torch.ones((latents.shape[2] - f_H, h * w // 4)) * timestep
  ```
- 通过 `inputs["mitty_human_frames"] = f_H` 传入 f_H
- 写 `MittyFlowMatchLoss`（基于 `FlowMatchSFTLoss` 修改）：
  ```python
  def MittyFlowMatchLoss(pipe, **inputs):
      human_lat = inputs["human_latents"]    # (1,48,f_H,h,w) clean
      robot_lat = inputs["input_latents"]    # (1,48,f_R,h,w) clean
      f_H = human_lat.shape[2]

      t = sample_t()
      noise = torch.randn_like(robot_lat)
      robot_noisy = pipe.scheduler.add_noise(robot_lat, noise, t)
      target = pipe.scheduler.training_target(robot_lat, noise, t)

      latents = torch.cat([human_lat, robot_noisy], dim=2)
      inputs["latents"] = latents
      inputs["mitty_human_frames"] = f_H  # 传给 patched model_fn
      # 复用 DiffSynth first_frame 机制：保证 human 段在 forward 里被再次替换为 clean
      inputs["first_frame_latents"] = human_lat

      noise_pred = pipe.model_fn(**models, **inputs, timestep=t)
      noise_pred_robot = noise_pred[:, :, f_H:]
      return mse(noise_pred_robot.float(), target.float())
  ```

### Step 2b: 训练脚本 `train_mitty.py`

**模型加载**（基于 DiffSynth `WanTrainingModule`，但替换 `task_to_loss["sft:train"]`）：
- 方案 1（推荐）：继承 `WanTrainingModule`，在 `__init__` 末尾覆盖 `self.task_to_loss["sft:train"] = MittyFlowMatchLoss` 并 monkey-patch `self.pipe.model_fn = mitty_model_fn_wan_video`
- 方案 2（备选）：完全自写 module，直接调 `WanVideoPipeline.from_pretrained` + inject_lora

LoRA 配置：
- `lora_base_model="dit"`, `lora_target_modules="q,k,v,o"`, `lora_rank=96`
- bf16，**不加** `fp8_models`

**CLI 参数**（基于 `train_lora.py` 改）：
- `--cache-train output/mitty_cache_1s/train` (required)
- `--cache-eval output/mitty_cache_1s/eval` (required, in-task eval)
- `--cache-ood output/mitty_cache_1s/ood_eval` (optional, OOD eval)
- 删除 `--eval-ratio`（split 已由目录分开）
- `--eval-video-samples-in-task` (默认 2)
- `--eval-video-samples-ood` (默认 2)
- 其他参数（--lora-rank、--lr、--epochs、--repeat、--save-steps、--eval-steps、--eval-video-steps）沿用 `train_lora.py`

**训练循环**：基本复用 `src/pipeline/train_lora.py` 的骨架（DDP helpers / 日志 / eval video / checkpoint），改动：
- `load_sample` 改成返回 `{human_latents, robot_latents, context_posi, context_nega}`
- model forward 改成走 `MittyFlowMatchLoss`
- Eval 阶段：分别对 eval_files 和 ood_files 算 loss，日志 `eval_loss_in_task=X eval_loss_ood=Y`
- `generate_eval_videos` 拆成 `eval/step-NNNN/in_task/` 和 `eval/step-NNNN/ood/` 两个子目录（传入两个 file list）

**Eval video 生成**（参考 `train_lora.py:197+`）：
- 取 eval 样本的 `human_latent` + text emb
- `robot_latent = randn(...)` 初始化
- Denoising loop：每步 concat + forward + CFG + step
- Eval 生成只取 `x[:, :, f_H:]` 作为 robot 输出
- VAE decode 并存为 mp4

### Step 3: 调参与验证

- 分辨率 480×640，帧数 17+17（human 17 + robot 17）→ VAE latent 5+5 = 10 temporal patches
- LoRA rank 96, lr 1e-4, AdamW, weight_decay 1e-2
- save-steps 50, eval-steps 50, eval-video-steps 100
- bf16, 无 FP8
- DDP 4 卡：`torchrun --nproc_per_node=4 -m src.pipeline.train_mitty ...`

## 验收标准

1. `python -m src.pipeline.mitty_cache --pair-dir training_data/pair/1s/train --output output/mitty_cache_1s/train` 能产出 143 个 .pth 缓存；`eval` 出 5 个，`ood_eval` 出 8 个
2. `torchrun --nproc_per_node=4 -m src.pipeline.train_mitty --cache-train ... --cache-eval ... --cache-ood ...` 能跑通至少 100 步（每 rank），等效 bs=4
3. 每步输出 `train_loss`（非 NaN），定期输出 `eval_loss_in_task` 和 `eval_loss_ood`，in-task loss 应明显下降
4. Eval 视频目录结构：`eval/step-NNNN/{in_task,ood}/{gt,ctrl,gen}_NN.mp4`，in-task 和 ood 各有样本
5. 单卡显存峰值 < 22 GB
6. 单步耗时可接受（预计 3~8 秒/step，依实测）

## 测试要求

- 跑 10 步 smoke test，确认 loss 合理（~0.1 量级，非 0/NaN）
- 跑 200 步完整训练，生成 2 个 eval checkpoint + 2 次 eval 视频（两种 split 都有）
- 检查 in-task eval generated 的 human 身体部位是否被替换成 robot（初期不期望效果很好，只要有 "robot-ish" 趋势即可）
- OOD eval 质量应明显差于 in-task（这是 OOD eval 的意义）
- 确认 LoRA checkpoint 可以保存为 safetensors

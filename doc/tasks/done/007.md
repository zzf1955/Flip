## 背景

Mitty LoRA 训练（T005）生成的视频中手部质量不够好。当前 `MittyFlowMatchLoss` 对 latent 空间所有位置施加均匀 MSE loss，手部区域仅占画面很小比例，梯度信号被背景稀释。

需要在 loss 中对手部区域施加更高权重，迫使模型更好地学习手部细节。

## 目标

1. 新建 `src/pipeline/hand_patch.py`：利用 FK 投影手部 mesh → 2D bbox → latent 空间权重图，独立于训练 pipeline
2. 修改 `MittyFlowMatchLoss`：支持可选 `patch_weights` 空间加权
3. 修改 `train_mitty.py`：加 `--patch-dir` 参数加载权重图
4. 提供 debug overlay 视频可视化手部 patch 区域

## 范围

**包含：**
- 新增 `src/pipeline/hand_patch.py`（权重图生成，含 debug overlay）
- 修改 `src/pipeline/mitty_model_fn.py`（加权 loss 分支）
- 修改 `src/pipeline/train_mitty.py`（加载权重图、传入 loss）
- 覆盖 train / eval / ood_eval 三个 split

**不包含：**
- 替换 patch 方案（本次用 FK mesh，后续可替换）
- BrainCo 手部支持（当前所有训练数据均为 Inspire 任务）
- 手部单独微调/冻结其他区域

## 实施计划（分步骤）

### Step 1: 新建 `src/pipeline/hand_patch.py`

**输入**：
- `--cache-dir`：mitty_cache 目录（读取 pair 文件名和 source_id）
- `--source-map`：source_map.json 路径
- `--output`：输出权重图目录
- `--hand-weight`：手部权重倍数（默认 3.0）
- `--margin`：像素空间 bbox 外扩（默认 20）
- `--debug-dir`：debug overlay 视频输出目录（可选）

**算法（每个 pair）**：
1. source_map → task/ep/seg/clip_idx
2. 加载 `training_data/segment/<task>/<ep>/<seg>_joints.parquet`
3. 提取 clip 对应帧（clip_idx × 30 ~ (clip_idx+1) × 30，基于 frame_index 偏移量 `seg_min + clip_idx * 30`）
4. 每帧：`build_q → do_fk → render_mask_for_links(left_hand) → mask_to_bbox`；右手同理
5. 所有帧 bbox 取 union → 单个 bbox per hand
6. 像素 bbox ÷16 → latent bbox（floor min, ceil max, clamp [0, 30)×[0, 40)）
7. 构造 weights tensor (5, 30, 40)：全 1.0，手部区域设为 hand_weight
8. 保存 .pth

**Debug overlay**：加载 robot segment 视频帧，绘制手部 mesh overlay + 16px latent grid + 高亮权重 cell，保存 MP4。

**输出格式**（接口契约，可替换）：
```python
{"weights": Tensor(5, 30, 40), "bboxes_pixel": {...}, "bboxes_latent": {...},
 "hand_weight": float, "source_id": str}
```

**复用模块**：
- `src.core.fk`: build_q, do_fk, parse_urdf_meshes, preload_meshes
- `src.core.config`: G1_URDF, MESH_DIR, BEST_PARAMS, get_skip_meshes
- `src.pipeline.sam2_segment`: BODY_PARTS, match_links, render_mask_for_links, mask_to_bbox

### Step 2: 修改 `src/pipeline/mitty_model_fn.py`

在 `MittyFlowMatchLoss` 中：
- line 148 后 pop `patch_weights`
- loss 计算增加分支：有 patch_weights 时用 `(mse_per_element * weights).mean()`，否则原 `F.mse_loss`
- 完全向后兼容

### Step 3: 修改 `src/pipeline/train_mitty.py`

- 新增 `--patch-dir` CLI 参数
- 训练循环：load_sample 后附带加载同名 patch .pth
- `collate_batch`：stack patch_weights
- `eval_loss`：自动推导 eval/ood patch 子目录
- 日志：启动时打印 patch weights 状态

### Step 4: 端到端验证

1. 对 eval split (4 pairs) 跑 hand_patch.py + --debug-dir
2. 目视检查 debug overlay
3. 对 train/eval/ood 三个 split 全量生成 weights
4. 跑 10 步 smoke test (有/无 --patch-dir)，确认 loss 值有差异
5. 跑一轮完整训练，对比 eval 视频手部质量

## 验收标准

1. `hand_patch.py` 成功为 132 个 pair 生成权重图 .pth
2. Debug overlay 视频中手部区域框选正确（与 FK overlay 对应）
3. 有 `--patch-dir` 时训练 loss 值与无 patch 时不同
4. 不传 `--patch-dir` 时训练行为与之前完全一致（向后兼容）
5. 单独运行 `hand_patch.py` 耗时 < 1 分钟

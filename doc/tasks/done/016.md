## 背景

`src/pipeline/make_robot_pair.py` docstring 声明是 "robot→robot reconstruction pairs"，但实际实现是**相邻 1s clip 配对**：每个 4s segment 切 4 个 clip，配出 (c0,c1) (c1,c2) (c2,c3) 三对作为 (control_video, video)。

经 `output/mitty_cache_robot/eval/pair_0000.pth` 验证：
- `human_latent` vs `robot_latent` 平均 |diff| ≈ 0.57
- `human_frames` vs `robot_frames` 像素 |diff| ≈ 31~47
- training run `2026-04-20_004842` 的 eval ctrl_*.mp4 和 gt_*.mp4 内容明显不同

整个"重建"任务其实在做"前 1s 预测后 1s"，违背 stage1 robot-recon 的本意。

## 目标

1. `make_robot_pair.py` 修为 identity 配对：每个 segment 生成 4 个 (c{i}, c{i}) 自重建 pair
2. metadata.csv 中同一行的 `video` 和 `control_video` 内容一致
3. 重生后的 cache 中 `human_latent ≈ robot_latent`、`human_frames == robot_frames`
4. 训练 step-0001 的 eval `ctrl_00.mp4` 与 `gt_00.mp4` 像素一致

## 范围

**包含：**
- 修 `src/pipeline/make_robot_pair.py`
- 备份旧的 `training_data/robot_pair/`（重命名为 `robot_pair_neighbor_bak`）和 `output/mitty_cache_robot/`
- 用小集（`--max-segments 10 --per-task-eval 1`）重生 pair + cache 验证
- 跑 1 个 short train run 验证 eval 视频

**不包含：**
- 全量 cache 重生（10000 sample，单卡十几小时，留给用户后续单独跑）
- `make_pair.py` 不动（FLIP 原版 human-robot pair 不受影响）
- `mitty_cache.py`、`train.py` 不动

## 实施计划（分步骤）

### Step 1: 改 `make_robot_pair.py`

- `make_pairs_from_segments`：循环 0~3，pair 字段 `clip` 和 `start = ci * 1.0`，source_id `..._c{ci}`
- `process_pair`：先 `cut_clip` 出 tgt mp4，再 `shutil.copyfile` 到 ctx 路径（两份完全相同）
- 给 source_map 字段去掉 `ctx_clip / tgt_clip`，加 `clip`

### Step 2: 备份 + 小集重生

```bash
mv training_data/robot_pair training_data/robot_pair_neighbor_bak
mv output/mitty_cache_robot output/mitty_cache_robot_neighbor_bak

python -m src.pipeline.make_robot_pair --task inspire \
  --max-segments 10 --per-task-eval 1 --workers 8

LD_PRELOAD=... python -m src.pipeline.mitty_cache \
  --pair-dir training_data/robot_pair/1s/train \
  --output output/mitty_cache_robot/train --device cuda:0
LD_PRELOAD=... python -m src.pipeline.mitty_cache \
  --pair-dir training_data/robot_pair/1s/eval \
  --output output/mitty_cache_robot/eval --device cuda:0
```

预期：5 tasks × 10 segs × 4 pair = 200，eval 20, train 180。

### Step 3: 静态校验 cache

python 脚本断言 `lat_max_diff < 1e-3` 且 `frame_mean_diff < 1.0`。

### Step 4: 短训练验证

```bash
torchrun --nproc_per_node=4 -m src.pipeline.train \
  --backbone mitty --loss uniform \
  --cache-train output/mitty_cache_robot/train \
  --cache-eval  output/mitty_cache_robot/eval \
  --epochs 1 --eval-steps 1 --eval-video-steps 1 \
  --eval-video-samples-in-task 2 --max-eval-files 4 \
  --wandb-tags stage1 robot-recon-identity verify
```

校验 step-0001 的 ctrl/gt mp4 像素 |diff| ≈ 0。

## 验收标准

1. metadata.csv 中 `video == control_video` 内容相同（hash 比较）
2. cache .pth: `(human_latent - robot_latent).abs().max() < 1e-3`
3. 训练 eval step-0001: `ctrl_00.mp4` 和 `gt_00.mp4` pixel |diff| ≈ 0
4. 旧的相邻 clip pair / cache 已备份，可恢复

## 测试要求

- 静态校验脚本通过（assert 不抛错）
- 一个短训练 run 完成，eval 视频对比通过

## 背景

当前 `src.pipeline.train` 的启动参数过多，数据 cache、T5 cache、loss、eval、LoRA merge 等配置散落在命令行和文档中。正式训练时需要手动传多段固定路径，容易传错，也不利于 W&B 中按参数检索 run。

## 目标

1. 将训练数据路径、eval/OOD cache、T5 cache 和默认输出目录集中到训练配置模块。
2. 以 `--task-name` 选择固定训练数据配置，CLI 只覆盖训练/评估/LoRA/W&B 参数。
3. 移除当前未使用的 `--loss` 入口，不保留 hand-patch loss 公开分支。
4. 保留多 LoRA merge，并自动检测被 merge checkpoint 的 rank。
5. 保留可训练 LoRA 参数覆盖能力。
6. 将 `max_steps` 默认设为 1000，save/eval/eval-video 默认设为 100，batch size 与 eval 参数默认可覆盖。
7. W&B tags 写入所有最终生效参数。

## 范围

包含：
- `src/pipeline/train.py` 训练入口 CLI 和配置解析。
- 新增训练 task 配置模块。
- LoRA merge rank 自动检测。
- W&B tags 构造。
- 训练文档、requirement log、launcher help 和 smoke 脚本更新。

不包含：
- 重新设计 Mitty loss/model_fn。
- 迁移历史 checkpoint/cache。
- 修改 main 上已有未提交脚本实验逻辑。

## 实施记录

- 新增 `src/pipeline/train_config.py`，以 `--task-name` 固定解析 train/eval/OOD cache、T5 cache 和输出目录。

- `src/pipeline/train.py` 默认 `--max-steps 1000`，`--save-steps 100`，`--eval-steps 100`，`--eval-video-steps 100`。

- 更新训练、FFN merge、W&B、two-stage、Codex migration、requirement log、launcher help 和 GPU smoke 脚本。

## 背景

当前仓库同时保留了多轮实验路径：Wan 2.1 FunControl、Rectified Flow、早期 DiffSynth 训练入口、Mitty 主线、三阶段 LoRA、多个数据生成脚本和历史文档。用户要求进入 develop 流程做一次重构：去掉 Fun Control、Dxxx/RectFlow 等废弃方法，重点规范数据管理、命名规范和配置约定；测试与临时产物统一放到 `./tmp`，卡 2 留给 Codex 验证，卡 3 留给用户实验。

本任务不沿用旧 T031；T031 已取消。

## 目标

1. 收敛正式训练/数据入口，删除或归档 FunControl、RectFlow、Dxxx Flow、直接替换噪声等废弃路径。
2. 避免新增对外部 DiffSynth 的业务耦合；训练主线优先使用本仓库 `src/core/wan_loader.py` 等直接加载/工具模块。
3. 建立清晰的数据目录约定：共享数据、worktree 输出、cache、日志、测试临时产物分层明确。
4. 统一命名与配置：CLI 参数、目录名、文档命名使用同一套主线词汇。
5. 更新文档，明确保留入口、废弃入口、GPU 分配与验证命令。

## 范围

**包含：**

- 清理 `src/pipeline/` 中废弃训练入口和 backbone 注册：FunControl / RectFlow / Dxxx Flow / 直接替换噪声。
- 将保留训练入口限定为 Mitty 外观替换与三阶段 LoRA 主线；必要时重命名或补充说明。
- 梳理 `src/core/config.py` 中路径常量，新增 `TMP_DIR` 等统一临时目录，并将轻量测试/烟测默认输出改到 `./tmp`。
- 更新 `doc/` 中训练、数据、配置相关文档与 `doc/requirement-log.md`。
- 运行不占用卡 3 的轻量验证；如需 GPU，显式使用 `CUDA_VISIBLE_DEVICES=2` 或 `scripts/flip_run.sh --cuda 2` 约束。

**不包含：**

- 不重做 SAM2 / retarget / inpaint / 渲染算法。
- 不迁移历史 checkpoint/cache；只文档说明旧产物不兼容或需重建。
- 不依赖外部 DiffSynth 作为新重构后的主线接口；遗留文档只作为历史记录。
- 不使用卡 3。
- 不在 main 分支直接做代码修改。

## 重构方案

### 1. 入口收敛

- 保留：`mitty_cache.py`、Mitty/Wan2.2 训练主线、三阶段 LoRA 相关脚本。
- 移除/归档：`train_lora.py`（Wan2.1 FunControl legacy）、`train_rf.py`、`rf_model_fn.py`、`backbones/rectflow.py`、统一训练入口中 `--backbone rectflow` 暴露。
- 对历史任务文档保持为完成记录，但当前文档和脚本索引不再推荐这些入口。

### 2. DiffSynth 边界

- 新代码不从外部 DiffSynth 训练框架 import 业务组件。
- 如短期仍需 DiffSynth 的底层类或调度器，集中在少数兼容模块，并在文档标注待替换边界；不让 pipeline 互相反向依赖 legacy 脚本。
- 优先复用/扩展 `src/core/wan_loader.py`、`src/core/train_utils.py`、`src/pipeline/mitty_model_fn.py`。

### 3. 数据与临时目录规范

- `data/`：只读原始/标定/机器人数据。
- `training_data/`：可复现实验数据、pair、cache、log。
- `output/`：人工检查和 pipeline 中间产物。
- `tmp/`：测试、smoke、一次性调试产物；默认可删除，不作为实验结果。
- 文档要求所有测试命令默认输出到 `./tmp/<module>/...`。

### 4. 命名规范

- 方法命名：`mitty` / `lora_stage{1,2,3}`；不再使用 `fun-control`、`rectflow`、`rf` 作为正式 tag。
- 数据 split：`train` / `eval` / `ood_eval`。
- cache：`training_data/cache/{vae,t5}/<dataset>/<split>`。
- 运行日志：`training_data/log/<method>-<task>-...`。

### 5. 验证策略

- CPU/轻量：import smoke、CLI `--help`、路径解析单测/脚本检查。
- GPU/可选：只用卡 2，运行小规模 cache 或训练 dry-run；不触碰卡 3。
- 所有验证输出到 `./tmp/t032/`。

## 验收标准

1. 正式入口和文档中不再暴露 FunControl / RectFlow / Dxxx Flow 作为可用新方法。
2. 测试、smoke、临时输出默认指向 `./tmp`。
3. 配置文档说明共享数据、worktree 输出、cache、tmp 的职责边界。
4. 新增或保留的主线代码不新增外部 DiffSynth 业务依赖。
5. 在卡 2 或 CPU 上完成轻量验证并记录结果。

## 测试要求

1. `python -m compileall src` 或等价 import smoke。
2. 保留 CLI 的 `--help` 检查。
3. 若执行 GPU 验证，使用卡 2 并写入 `./tmp/t032/`。

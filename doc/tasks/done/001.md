## 背景

当前 `src/core/config.py:10` 用 `BASE_DIR = 两级上跳 __file__` 派生所有路径，`data/`、`weights/`、`paper/`、`ref-cosmos-*/`、`ProPainter/` 都拼接为 `BASE_DIR/<name>`。这些目录都在 `.gitignore`，所以 `git worktree add` 不会复制它们——但 worktree 下 `BASE_DIR` 指向 worktree 自身，相应目录**根本不存在**，脚本会直接崩。

涉及的大目录体量：
- `data/` 95 GB（数据集 + mesh + URDF + 标定，已 gitignore）
- `ref-cosmos-transfer2.5/` 13 GB（已 gitignore）
- `ref-cosmos-transfer1/` 383 MB（已 gitignore）
- `ProPainter/` 318 MB（已 gitignore）
- `weights/` 191 MB（已 gitignore）

总计 ~109 GB，多 worktree 各自复制不现实，必须共享。

**注意**：`paper/`（922 MB）已被 git 追踪（commit `3032bef`），本任务**不纳入共享规范**——worktree 会通过 checkout 各自拿到一份，硬盘够用可接受。

另外发现 `data/output/` 下有 566 MB 旧产物残留（`inpaint/`、`inpaint_v2/`、`mask/`），grep 确认 `src/**` 无任何引用，属于旧 pipeline 遗留，应清理。

## 目标

1. 所有只读大目录从固定绝对路径 `/disk_n/zzf/flip/<name>` 读取，worktree 开多少个都不复制、不占空间；
2. `output/` 保持 per-worktree 隔离，不同分支跑实验不互相覆盖；
3. 规范写进 `CLAUDE.md`：禁止拼 `BASE_DIR/data` 之类相对路径，统一走 `config.*_ROOT`；
4. 新建 worktree 后无需手动配置，`python -m src.pipeline.*` 能直接跑；
5. 清理 `data/output/` 旧产物。

## 范围

**包含**
- `src/core/config.py` 增加 `MAIN_ROOT / DATA_ROOT / WEIGHTS_ROOT / COSMOS1_ROOT / COSMOS25_ROOT / PROPAINTER_ROOT` 常量（支持 `FLIP_MAIN_ROOT` 环境变量覆盖）
- `OUTPUT_DIR` 明确注释为"per-worktree，保留 `BASE_DIR/output`"
- 扫描并改造 `src/**` 中所有直接拼接这些目录的地方
- 更新 `CLAUDE.md`，在"代码结构"或单开一节写"路径规范"
- 删除 `/disk_n/zzf/flip/data/output/`
- 可选：`scripts/setup_worktree.sh` 做路径自检

**不包含**
- worktree 创建流程自动化（不写 git hook）
- `ref-cosmos-*` 的代码改造（只改本仓库里怎么引用它们的路径）

## 实施计划（分步骤）

1. **config 改造** — `src/core/config.py`
   - 新增常量：
     ```python
     MAIN_ROOT = os.environ.get("FLIP_MAIN_ROOT", "/disk_n/zzf/flip")
     DATA_ROOT = os.path.join(MAIN_ROOT, "data")
     WEIGHTS_ROOT = os.path.join(MAIN_ROOT, "weights")
     COSMOS1_ROOT = os.path.join(MAIN_ROOT, "ref-cosmos-transfer1")
     COSMOS25_ROOT = os.path.join(MAIN_ROOT, "ref-cosmos-transfer2.5")
     PROPAINTER_ROOT = os.path.join(MAIN_ROOT, "ProPainter")
     ```
   - `_MESH_ROOT` / `DATASET_ROOT` 改为基于 `DATA_ROOT`
   - `OUTPUT_DIR` 保留 `BASE_DIR/output` 并加注释说明隔离原因
   - 启动时对 `DATA_ROOT` 做存在性检查，不存在就 `raise RuntimeError` 提示设置 `FLIP_MAIN_ROOT`

2. **全局扫描替换** — `src/**`
   - `grep -rn "BASE_DIR.*data\|/data/\|/weights/\|/paper/\|ref-cosmos\|ProPainter" src/` 找出所有硬编码
   - 统一改为 `config.DATA_ROOT / config.WEIGHTS_ROOT` 等
   - 特别检查 `src/core/fk.py`、`src/core/data.py`、`src/core/smplh.py`

3. **规范落地** — `CLAUDE.md`
   - 新增一节"路径规范（worktree 兼容）"，写明：
     - 只读共享目录列表 + 绝对路径
     - 必须走 `config.*_ROOT`，禁止拼 `BASE_DIR/data`
     - `output/` 是 per-worktree 的产物，不共享
     - `FLIP_MAIN_ROOT` 环境变量用法

4. **清理旧产物**
   - `rm -rf /disk_n/zzf/flip/data/output`（先确认 grep 结果真的无引用再删）

5. **自检脚本（可选）** — `scripts/setup_worktree.sh`
   - 检查 `FLIP_MAIN_ROOT` 下各大目录可达
   - 打印当前 worktree 路径和共享路径

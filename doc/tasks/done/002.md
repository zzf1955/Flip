## 背景

Cosmos Transfer 2.5 在共享机器上因 CPU RAM 不足（其他用户占 58GB）反复触发 OOM killer，且 guided generation 效果不理想（mask 外背景也被影响）。需要更轻量、mask 更严格的替代方案。

## 目标

1. 下载 Wan 2.1 VACE 1.3B 模型到 ComfyUI
2. 编写 `wan_regen.py` 脚本调用 ComfyUI API，喂入 `cosmos_prepare` 的输出（depth + mask + composite）
3. 单卡 4090 跑通 16 帧测试，验证：depth 控制有效、mask 保护背景、生成质量可接受

## 范围

**包含：**
- 模型下载与配置
- `src/pipeline/wan_regen.py` 新建
- `src/core/config.py` 新增 ComfyUI 相关常量
- 端到端测试 (Pickup_Pillow ep0)

**不包含：**
- ComfyUI 本身的修改
- 批量生成 pipeline
- 14B 模型测试（后续任务）

## 实施计划（分步骤）

1. 下载 3 个模型文件 (~9.5GB)：VACE 1.3B、umt5_xxl_fp8、wan_2.1_vae
2. config.py 新增 COMFYUI_ROOT、WAN_REGEN_DIR
3. 编写 wan_regen.py：读取 prepare 输出 → 构建 ComfyUI workflow → API 调用 → 后处理合成
4. 启动 ComfyUI (GPU 2) 测试模型加载
5. 端到端测试 16 帧
6. 视觉检查输出

## 验收标准

- [x] 3 个模型文件就位（`wan2.1_vace_1.3B_fp16.safetensors` + `umt5_xxl_fp8_e4m3fn_scaled.safetensors` + `wan_2.1_vae.safetensors`，位于 `/disk_n/zzf/ComfyUI/models/`）
- [x] ComfyUI 能加载 Wan 2.1 VACE 模型
- [x] wan_regen.py 产出 output 视频（webm，640×480, 17 frames @ 16 fps）
- [x] 背景区域像素与原始一致（mask 生效）
- [x] 人体区域有自然纹理（depth 引导有效）

## 测试要求

- 输入：cosmos_prepare 输出 (Pickup_Pillow_MainCamOnly_ep0_s5_d1)
- 输出：output/human/wan_regen/ 下生成视频
- VRAM 占用 <16GB

### 路径决策

原目标是只做 Wan VACE；实际流程为"先 Cosmos Transfer 2.5 失败 → 改 Wan VACE"。两个方案共享前置 `cosmos_prepare.py`（名字保留，因其生成的 depth/mask/composite 对两条路径都适用）。

| 方案 | 脚本 | 结果 |
|------|------|------|
| **Cosmos Transfer 2.5** | `src/pipeline/cosmos_regen.py` | 跑通但弃用 — 共享机 CPU RAM OOM（2/3 GPU 多次重试）+ mask 外背景被改 |
| **Wan 2.1 VACE 1.3B（via ComfyUI）** | `src/pipeline/wan_regen.py` | 采用 — 单卡 4090，mask 严格，原生 depth 控制 |

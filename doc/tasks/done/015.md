## 背景

当前训练 pipeline 有两个几乎完全重复的脚本：

- `src/pipeline/train_mitty.py`（743 行）— Mitty in-context：`cat([clean_human, noisy_robot])`，per-patch timestep，robot 段 loss
- `src/pipeline/train_rf.py`（625 行）— Rectified Flow：`noisy = (1-σ)*target + σ*source`，source 代替 Gaussian noise，全帧 loss

两者差异只有 5 处：`TrainingModule` 里装的 `model_fn`/Loss、`generate_eval_videos` 里 denoise 内循环 20 行、logger name、W&B tag、argparse description。其余 90% 完全相同，`train_rf.py` 还反向 import 了 `train_mitty.py` 的 `build_pipe` / `prepare_sample` / `collate_batch` / `MANUAL_DIR` / `DEFAULT_*` 等 helpers。

hand_patch loss 是正交的第二维度：两个 Loss 函数里都带 `if patch_weights is not None` 分支，当前靠「有没有 `--patch-dir`」隐式触发。

做消融实验时需要 2×2 四组合（mitty/rectflow × uniform/hand_patch），但目前必须在两个脚本之间手动切换，CLI 不一致、W&B 对比困难。

## 目标

1. 新增统一入口 `src/pipeline/train.py`，通过 `--backbone {mitty,rectflow}` + `--loss {uniform,hand_patch}` 一行命令切换消融维度
2. 显式 `--loss` 取代隐式 `patch_dir`：互斥校验（`--loss uniform --patch-dir X` 或 `--loss hand_patch` 无 `--patch-dir` 直接 `ap.error`）
3. W&B tag 自动 = `[backbone, loss] + wandb_tags`，消融表按 tag 分面一目了然
4. 旧脚本 `train_mitty.py` / `train_rf.py` **完全保留不动**（用户明确要求，由新入口反向 import 复用）
5. 数值等价性：四组合 × 同 seed 跑同样 step 数，train_loss 与旧脚本严格一致

## 范围

### Step 2: `src/pipeline/train.py` 统一入口

基于 `train_mitty.py` 改造，改动：

1. 顶部加 `from src.pipeline.backbones import get as get_backbone, all_names`
2. 顶部 `from src.pipeline.train_mitty import build_pipe, _load_patch_weights, prepare_sample, collate_batch, MANUAL_DIR, DEFAULT_DIT_DIR, DEFAULT_VAE, DEFAULT_TOKENIZER`（复用 helpers，不重写）
3. `MittyTrainingModule` import 和 `mitty_model_fn`/`MittyFlowMatchLoss` import 移除
4. `generate_eval_videos(spec, model, files, ...)` — 签名加 `spec`，内部 denoise 循环（原 20 行）替换为 `denoised = spec.eval_denoise_fn(pipe=pipe, sample=s, sched=sched, device=device, cfg_scale=cfg_scale, num_inference_steps=num_inference_steps)`；外壳（VAE decode + save mp4）保留
5. `train(args)` 内三处替换：
   - `log = setup_logging(..., name="train_mitty")` → `name=spec.log_name`
   - `tags=(args.wandb_tags or []) + ["mitty"]` → `+ [spec.wandb_tag, args.loss]`

7. `main()` 新增 argparse：
   - `--backbone`（choices=all_names()，default="mitty"）
   - `--loss`（choices=["uniform","hand_patch"]，default="uniform"）
   - 解析后两条校验：
     - `if args.loss == "hand_patch" and not args.patch_dir: ap.error("--loss hand_patch requires --patch-dir")`
     - `if args.loss == "uniform" and args.patch_dir: ap.error("--patch-dir only valid with --loss hand_patch")`
   - `spec = get_backbone(args.backbone)` 后 `train(args, spec)` 传入
8. argparse `description` 用通用文案 `"Unified LoRA training for Wan 2.2 TI2V-5B"`；spec.description 在启动日志里 info

### 使用方式

消融实验一行命令切换：

```bash
python -m src.pipeline.train --backbone mitty    --loss uniform    ...
python -m src.pipeline.train --backbone mitty    --loss hand_patch --patch-dir ... ...
python -m src.pipeline.train --backbone rectflow --loss uniform    ...
python -m src.pipeline.train --backbone rectflow --loss hand_patch --patch-dir ... ...
```

W&B tag 自动 `[backbone, loss]`。`--seed` 生效的 deterministic loss 曲线（step=1 精确重现）。

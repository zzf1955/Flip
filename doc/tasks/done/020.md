## 背景

训练用 `torchrun --nproc_per_node=2` 启动 DDP。DiT safetensors 是 FP32 存储（5B × 4 = 20GB），每次加载时逐 tensor `.to(bf16)` 转换。多 rank 并发/串行读取 20GB × N 极慢（当前 rank-sequential 方案仍需 N 次磁盘读取）。

实测数据：
- DiT: 3 分片, 825 tensors, 全部 FP32, 共 20GB
- VAE: FP32, 2.82GB
- T5: 已经是 bf16, 11.36GB（训练时不加载，embeddings 预缓存）

## 目标

1. DiT FP32 分片预转为 bf16 单文件 safetensors（~10GB），减少磁盘读量 50%
2. DDP 训练时 rank 0 独占读盘 + NCCL broadcast 到其他 rank，磁盘读次数 N → 1
3. 单 GPU 训练行为不变

## 范围

**包含**：
- 预转 bf16 脚本
- `wan_loader.py` 的 `load_dit` / `build_dit_shard_list` 修改
- `train_mitty.py` / `train_rf.py` 穿透 `skip_dit_load` 参数
- `train.py` broadcast 替换 rank-sequential

**不包含**：
- VAE 加载优化（已经只在 rank 0 加载）
- T5 优化（训练时不加载）
- 旧版 legacy 训练脚本中的独立训练循环

## 实施计划（分步骤）

### Step 1: 预转 bf16 脚本

新建 `src/tools/convert_dit_bf16.py`：读取 3 个 FP32 分片 → 合并 → 转 bf16 → 存为单个 `diffusion_pytorch_model-bf16.safetensors`。

### Step 2: `wan_loader.py` 修改

- `build_dit_shard_list()`: 优先检测 bf16 单文件
- `load_dit()`: 加 `skip_load=False` 参数，`True` 时用 `to_empty()` 分配空壳

### Step 3: `train_mitty.py` 穿透

- `build_pipe()` 加 `skip_dit_load=False`
- `MittyTrainingModule.__init__()` 加 `skip_dit_load=False`

### Step 4: `train_rf.py` 穿透

- `RFTrainingModule.__init__()` 加 `skip_dit_load=False`

### Step 5: `train.py` broadcast

- 删除 `for loading_rank in range(world_size)` 循环
- 删除旧的 LoRA-only broadcast
- 新增: `skip_dit_load=(rank != 0)` + `model.pipe.dit` 全参数 broadcast

## 验收标准

1. bf16 文件存在且可正常加载
2. 单 GPU 训练行为不变
3. DDP 训练日志显示 rank 0 从磁盘加载、其他 rank allocate empty + broadcast
4. 训练 loss 与修改前一致（相同 seed、相同步数）

## 测试要求

1. 单 GPU 冒烟测试（3 steps）
2. DDP 2 GPU 冒烟测试（3 steps）
3. 对比修改前后 loss 一致性

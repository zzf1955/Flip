## 背景

训练代码存在两套不一致的 run 命名方案：旧脚本用 `build_run_name()` 生成 `{prefix}-r{rank}-lr{lr}-{timestamp}`，新统一入口 `train.py` 仅用纯时间戳。W&B tags 缺少 loss 类型、数据规模等关键信息。每次跑实验需要手动指定 `--wandb-run-name`，否则在 W&B 面板上无法区分 run。

## 目标

- 本地 log 目录名 = W&B run name，自动生成，包含 backbone、lora rank、max steps、数据规模、日期
- W&B tags 自动记录完整超参信息
- 4 个训练脚本统一命名规范

## 范围

**包含：**
- `src/core/train_utils.py`：重写 `build_run_name()` 和 `build_wandb_tags()`
- `src/pipeline/train.py`、`train_mitty.py`、`train_rf.py`、`train_lora.py`：统一调用

**不包含：**
- 训练逻辑本身不变
- 不删除旧脚本

## 实施计划（分步骤）

1. 重写 `train_utils.py` 中的 `build_run_name()` 和 `build_wandb_tags()`，新增 `count_cached_files()`
2. 更新 `train.py`：run_name 移到数据加载后，调用新函数
3. 更新 `train_mitty.py`：同上
4. 更新 `train_rf.py`：同上
5. 更新 `train_lora.py`：同上

## 验收标准

- 所有脚本的本地目录名和 W&B run name 一致
- run name 格式：`{backbone}[-{loss}]-r{rank}-{steps}s-{n_train}d-{MMDD_HHMM}`
- W&B tags 包含：backbone, loss, rank, lr, bs, steps, data count, warmup, targets, gpu count

## 测试要求

- 检查 `build_run_name` 和 `build_wandb_tags` 输出格式正确
- 4 个脚本的 `--help` 正常

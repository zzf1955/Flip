## 背景

当前 `robot_patch.py` 使用 FK mesh 光栅化生成机器人 mask（`render_mask()`），精度受限于相机标定和 mesh 精度，边界粗糙。项目已有成熟的 SAM2 分割流程（`sam2_inpaint.py`、`segment_pipeline.py`），可产生 pixel-level 精确 mask。

## 目标

1. 新增 SAM2 mask 预计算 pipeline，对所有 14270 segment 生成高质量 mask
2. 修改 robot_patch.py 支持从预计算 SAM2 mask 加载，替代 FK mesh mask
3. 多 GPU 并行调度，4 GPU 6-10 小时完成全量预计算

## 范围

**包含：**
- 新增 `src/pipeline/sam2_precompute.py`（单 GPU SAM2 预计算）
- 新增 `src/pipeline/batch_sam2_precompute.py`（多 GPU 调度）
- 修改 `src/pipeline/robot_patch.py`（添加 `--mask-source sam2` 路径）

**不包含：**
- 不修改 `sam2_inpaint.py`、`segment_pipeline.py` 等现有 pipeline
- 不修改训练代码

## 实施计划（分步骤）

### Step 1: 创建 sam2_precompute.py
- 复用 segment_pipeline.py 的 SAM2 分割逻辑（BODY_PARTS、bbox prompt 生成、propagation）
- 复用 robot_patch.py 的 segment 发现逻辑（find_segments）
- 实现 compute_segment_mask()：视频读取 → FK prompt → SAM2 propagation → npz 保存
- 支持分片处理（--shard-index/--num-shards）和 resume

### Step 2: Smoke test
- 跑 2-3 个 segment 验证 mask 质量
- 检查 npz 文件格式和大小

### Step 3: 创建 batch_sam2_precompute.py
- 多 GPU subprocess 调度（参照 batch_inpaint.py）
- 日志和进度报告

### Step 4: 修改 robot_patch.py
- 添加 --mask-source {fk, sam2} 参数
- sam2 模式从 npz 加载 mask，跳过 FK 计算
- 保持 FK 模式完全不变

### Step 5: 端到端测试
- SAM2 mask → robot_patch → 比较输出质量

## 验收标准

1. `sam2_precompute.py` 能正确生成 (120, 480, 640) uint8 mask npz 文件
2. `batch_sam2_precompute.py` 能在 4 GPU 上并行运行
3. `robot_patch.py --mask-source sam2` 产出与 FK 模式格式兼容的训练数据
4. SAM2 mask 边界质量明显优于 FK mesh mask

## 测试要求

- smoke test: 2-3 个 segment 的 SAM2 预计算
- 集成测试: robot_patch.py --mask-source sam2 产出完整 pair 数据
- 质量对比: FK vs SAM2 mask 的 degraded video 视觉对比

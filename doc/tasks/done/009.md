## 背景

训练 pipeline（train_lora.py / train_mitty.py）每隔 N 步输出 eval 视频（gt_XX.mp4, ctrl_XX.mp4, gen_XX.mp4），分 in_task / ood 两个 split。目前只能肉眼看视频或看 eval loss 曲线，缺少标准重建指标的定量评估工具。

T006 计划中只覆盖了 PSNR/SSIM/MSE，且明确把 FVD 放到二期。用户要求更全面的指标覆盖。

## 目标

新建 `src/tools/eval_metrics.py`，一键扫描训练 run 的 eval 目录，对 gen vs gt 视频对计算多个重建指标，输出 CSV + 终端汇总表。

指标覆盖：
| 指标 | 类型 | 方向 | 实现方式 |
|------|------|------|----------|
| PSNR | 逐帧像素 | ↑ | skimage |
| SSIM | 逐帧结构 | ↑ | skimage |
| LPIPS | 逐帧感知 | ↓ | VGG16 自实现（环境无 lpips 包） |
| FID | 帧级分布 | ↓ | InceptionV3 + scipy sqrtm |
| FVD | 视频级分布 | ↓ | InceptionV3 mean-pool + scipy sqrtm |

## 范围

**包含：**
- `src/tools/eval_metrics.py`：独立 CLI 工具
- 读视频：ffmpeg pipe → numpy (T,H,W,3)
- LPIPS：VGG16 5 层特征 normalized L2（等价于 lpips 包 vgg 模式，无 learned linear weights）
- FID：InceptionV3 pool3 2048-d 特征 → Gaussian Fréchet distance
- FVD：每个视频的 Inception 特征 mean-pool → 视频级 Fréchet distance（样本数 ≥3 时才算）
- CLI：`--run`, `--split`, `--steps`, `--device`, `--no-lpips`, `--no-fid`, `--csv`
- 输出：`<run>/eval_metrics.csv` + 终端 summary table

**不包含：**
- 集成到训练循环中（保持独立工具）
- W&B 上传指标（后续可做）
- 新增 pip 依赖

## 实施计划（分步骤）

### Step 1: 开发 `src/tools/eval_metrics.py`

核心模块：
1. `read_video_frames(path)` — ffmpeg → numpy (T,H,W,3)
2. `LPIPS(nn.Module)` — VGG16 5 层特征提取 + normalized L2
3. `InceptionFeatureExtractor(nn.Module)` — InceptionV3 → 2048-d
4. `frechet_distance(mu1, sigma1, mu2, sigma2)` — Gaussian FD via scipy sqrtm
5. `compute_pairwise_metrics()` — 逐帧 PSNR/SSIM/LPIPS
6. `compute_fid()` — 帧级 FID
7. `compute_fvd()` — 视频级 FVD
8. `main()` — CLI 入口，扫描 eval 目录，汇总输出

复用 `src/core/train_utils.py` 的 `FFMPEG` 常量。

### Step 2: 在最近训练 run 上验证

```bash
python -m src.tools.eval_metrics --run 2026-04-18_163933 --split in_task --steps 0200 --device cuda:2
python -m src.tools.eval_metrics --run 2026-04-18_163933 --device cuda:2
```
